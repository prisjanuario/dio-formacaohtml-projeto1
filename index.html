1.1 Primeiros computadores

Pode-se dizer que o primeiro computador foi inventado pelo francês Blaise Pascal, durante o século XVII. Essa máquina era, na verdade, uma calculadora que realizava operações de soma e subtração. Mais tarde, o alemão Gottfried Wilhelm criou a possibilidade de realizar outras operações matemáticas, como a multiplicação e a divisão. Entrando no século XVIII, Ada Lovelace introduz o primeiro sistema de algoritmos do mundo, que passa a ser utilizado na Máquina Analítica, criado por Charles Babbage. Mais adiante, em 1943, é criada a primeira geração de computadores modernos, pelo inglês Alan Turing. Anos mais tarde, em 1981, a IBM lança o primeiro microcomputador pessoal.

1.2 Lado do cliente (client-side)

O conceito de client, ou cliente, diz respeito ao código que será processado apenas pelo navegador, ou browser do usuário. Por isso, também é chamado de front-end, e utiliza linguagens como o JavaScript. Trata-se da parte da interface, ou seja, tudo o que o usuário irá visualizar em sua tela.

< Voltar ao Índice

1.3 Lado do servidor (server-side)

O lado do servidor refere-se a todo o código que não será processado diretamente pelo navegador. Nesse caso, o navegador manda uma requisição para o servidor, que é o local onde o banco de dados fica armazenado, e essa requisição é processada no back-end da aplicação. Algumas linguagens utilizadas para desenvolver o server são PHP, Ruby, C# e JavaScript (Node.js).

2.1 Inspetor de elementos

O inspetor de elementos, como o nome já diz, serve para inspecionar os elementos de uma página web. Com ele, é possível ter acesso ao código-fonte e localizar elementos, como imagens, tags e outros. Também é possível alterar alguns elementos, como cores e textos. Todavia, essas alterações são feitas apenas no navegador, e não no código-fonte original da aplicação.

2.2 Estrutura básica do HTML

O HTML é estruturado pelos seguintes elementos:

Head: Contém a informação meta sobre o documento HTML. É aqui que inserimos coisas como o idioma e o título da página.

Title: Fica dentro do head. É o título da página que será exibido na aba da página.

Body: O body, ou corpo da página, é um contâiner que contém todo o conteúdo que será exibido para o usuário, como headings, parágrafos, imagens e links.

Headings e parágrafos: Indicam a hierarquia do conteúdo a ser exibido na página.

2.3 Tags e atributos

As tags servem para indicar a estrutura da página ao navegador. São uma forma de organizar os elementos presentes no documento HTML, de forma a facilitar sua interpretação pelo browser. As tags devem estar sempre dentro dos sinais matemáticos "menor que" e "maior que": < isto é um exemplo de tag >. Além disso, também devem possuir fechamento, ou seja, sempre que uma tag for aberta, ela deve ser fechada com uma barra (/). O conteúdo a ser exibido na página deverá ficar após a abertura e antes do fechamento da tag.

Já os atributos de tags são características que o conteúdo, a ser inserido dentro da tag, carrega. Exemplos de atributos são "style", que pode servir para alterar a cor, e "width", que serve para alterar a largura do contâiner onde o conteúdo está inserido. A sintaxe dos atributos constitui-se em descrevê-los dentro da própria tag, acompanhados por um sinal de igual e aspas, dentro das quais a característica do atributo será inserida.

2.4 Textos

A estrutura de um texto em um documento HTML é composta por headings e parágrafos, representados, respectivamente, pelas tags < h1 > (podendo ir até o < h6 >) e < p >. Já a formatação dos textos pode ser inserida através de tags como < i > para itálico, < b > para negrito, assim por diante.

2.5 Listas ordenadas e não ordenadas

As listas num documento HTML são classificadas em ordenadas e não ordenadas. As primeiras classificam o conteúdo em ordem e são inseridas através da tag < ol >, e geralmente utilizam números. Já as segundas, inseridas pela tag < ul > os classificam sem ordem de relevância, geralmente utilizando bullet points.

Esta é uma lista ordenada

  1. Item 1;
  2. Item 2;
  3. Item 3.

Esta é uma lista não ordenada

  • Item 1;
  • Item 2;
  • Item 3.

2.6 Links

Em um documento HTML, é possível adicionar hyperlinks que redirecionam o usuário para outra página web. Em sua sintaxe, os links são inseridos através da tag de ancoragem < a href="cliqueaqui.com" >. Também é possível adicionar atributos como < alt="Texto alternativo" >, para inserir o texto alternativo para fins de acessbilidade, < target="_blank" > para o link abrir em uma nova aba etc.

3.1 Criando um repositório no GitHub

As formações e Bootcamps da DIO são baseadas no desenvolvimento de projetos que, por sua vez, são adicionados ao repositório do GitHub e enviados para avaliação.

Passo a passo para criar um repositório

Passo 1

No canto superior direito da barra de menu do GitHub, próximo à foto de perfil, clicar no ícone "+" e ir em "Novo repositório".

Passo 2

Após ser direcionado para uma nova página, escolher o nome do repositório e digitar no campo indicado.

Passo 3

No passo 4, escolher se quer que o repositório seja público e privado.

Passo 4

Optar ou não por adicionar um README ao repositório. Trata-se de um arquivo que contém a descrição do projeto. Após isso, ir em "Criar repositório".

< Voltar ao Índice

3.2 Subindo o projeto no repositório

Passo 1

Para subir o projeto no repositório criado, é preciso primeiro instalar o Git na máquina.

Passo 2

Ir até a pasta em que o arquivo HTML está localizado e, com o botão direito do mouse, clicar em "Git Bash Here".

Passo 3

Após aberta a janela do Git, verificar se está na pasta certa do projeto que se pretende subir para o GitHub. Verificado isso, digitar o comando git init e apertar Enter. Depois, digitar git add nomedoarquivo.html. Em seguida, dar o comando git commit -m "nome do commit"

Fim. Obrigada!
